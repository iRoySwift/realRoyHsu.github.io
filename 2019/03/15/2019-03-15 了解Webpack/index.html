了解Webpack

本质上，webpack 是一个现代 JavaScript 应用程序的静态模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

1、熟悉 webpack

  • 创建项目文件（webpack）

    1. npm init;

      初始化 package.json

    2. 安装生产依赖 lodash

      npm install –save lodash

    3. 安装开发依赖 gulp

      npm install –save-dev gulp

2、开始安装配置 webpack

  • 创建 webpack.config.dev.js 配置文件

    1. 安装项目依赖

      npm install –save-dev webpack@4 或 npm i -D webpack

    2. 配置 webpack.config.js 文件前，path 入口，先要有项目文件夹 src

    3. 建 src 文件夹，放 App.js 项目入口,配置到 webpack.config.js 的入口中；output;项目的绝对路径，所以要 path 模块来获取;(dirname)当前文件所在的绝对路径;path.resolve([from …], to)

    4. 建立项目出口文件 dist 生成/assets，放置我们打包的文件

  • 修改 package.json 文件

  • 打包生产 main.js

    npm install webpack-cli -D

  • 运行 npm run dev,就会执行 package.json 里面的 “dev”: “webpack”
    生成 dist/assets/main.js 文件
  • 在 src 建立 index.html 文件，调用 main.js
